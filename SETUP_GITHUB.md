# GitHub Actions で「連続取得」をサーバーレスに回す手順

PCを使わず、GitHubの無料の自動実行（Actions）で、cronで少しずつJANを取得します。
取得結果の `jan_db.sqlite3` はリポジトリに保存され、次回はその続きから自動で再開します。

---

## 0. 仕組み（概要）
- 定期実行（既定：毎日 日本時間3:00）または手動実行で `bulk_import.py` が走る
- 1回あたり `LIMIT` 件だけ取得 → 更新したDBをリポジトリにコミット
- 「検索済み」は記録されるので、毎回続きから進む（=分割して少しずつ完成）

---

## 1. リポジトリを用意
1. https://github.com でアカウント作成（無料）
2. 「New repository」で新規作成。**Private（非公開）推奨**
   - Private: 取得データが非公開。ただし無料の自動実行は **月2,000分** まで
   - Public: 自動実行は **無制限（無料）** だが、コードとデータが公開される
3. このフォルダ（jancode-system）の中身をアップロード
   - GitHubの「Add file > Upload files」にドラッグ＆ドロップでも可
   - **`.env` はアップロードしないこと**（鍵が入っているため。`.gitignore` で自動除外されます）
   - `jan_db.sqlite3` は一緒に上げてOK（これまでの取得分を引き継げます）

---

## 2. 認証情報を登録（Secrets）
リポジトリの **Settings → Secrets and variables → Actions →「Secrets」タブ → New repository secret** で、
`.env` に入れている値を1つずつ登録します。

| Secret 名 | 値 |
|---|---|
| `RAKUTEN_APP_ID` | 楽天のアプリID(UUID) |
| `RAKUTEN_ACCESS_KEY` | 楽天のアクセスキー(pk_…) |
| `YAHOO_APP_ID` | Yahooのアプリ(Client)ID ※あれば |
| `APP_PUBLIC_URL` | `http://enzantrades.co.jp`（楽天の許可Webサイトと一致させる） |
| `OFF_CONTACT` | 連絡先メール（任意） |

---

## 3. スキャン対象を登録（Variables）
同じ画面の **「Variables」タブ → New repository variable** で、定期実行の対象を設定します。

| Variable 名 | 例 | 説明 |
|---|---|---|
| `BASE`  | `4901085` | GS1事業者コード(先頭桁)。対象メーカー |
| `START` | `0` | 開始アイテム番号 |
| `END`   | `100000` | 終了(含まない) |
| `LIMIT` | `2000` | 1回の最大件数（実行時間に直結） |
| `DELAY` | `1.5` | 1件あたり待機秒（楽天の制限に配慮） |

---

## 4. 動かす
1. リポジトリの **Actions** タブを開き、初回は「I understand…」で有効化
2. 左の「JANコード一括取得」→ **Run workflow** で手動テスト
   - base などを入力して実行 → ログに `[HIT] …` が出ればOK
3. 以後は cron（既定：毎日 日本時間3:00）で自動実行されます
   - 時間を変えたい場合は `.github/workflows/bulk_import.yml` の `cron:` を編集

> スケジュール実行はリポジトリが60日間まったく更新されないと自動停止します。
> このワークフローは毎回DBをコミットするので、稼働中は停止しません。

---

## 5. 取得データの確認・利用
- 取得が進むと、リポジトリの `jan_db.sqlite3` が更新されていきます
- 手元で見たいとき：`jan_db.sqlite3` をダウンロードして、ローカルの `jancode-system` フォルダに置き、
  `start.bat` を実行 → http://localhost:5057 で一覧・詳細・CSV出力ができます
- 画面もサーバー公開したくなったら、別途VM/PaaSに載せる構成に拡張できます

---

## 6. 実行時間とペースの目安
- 無料の自動実行は Private で **月2,000分**。1回 `LIMIT=2000`・`DELAY=1.5` だと約50分
  → 1日1回なら月約1,500分で収まります（2回/日にすると超過に注意）
- もっと速く回したい場合：
  - リポジトリを **Public** にする（実行時間は無制限・無料。ただしデータ公開）
  - または対象を **9桁ベース**（候補約1,000件）に絞ると1回で完了します
- 7桁ベース（候補10万件）は分割になるため日数がかかります。対象メーカーを絞るほど効率的です

## 注意
- 各APIの利用規約・レート制限を順守してください（`DELAY` を下げ過ぎない）。
- `.env` や鍵はコミットしないでください（Secretsで管理）。
